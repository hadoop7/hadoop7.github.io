Introduction

               Hadoop is an open source, Java-based programming framework that supports the processing and storage of extremely large data sets in a distributed computing environment. It is part of the Apache project sponsored by the Apache Software Foundation.

Hadoop needs Java to be installed in the system. If you dont have java installed in the system, follow from this step , or if you hav java installed in your system, you cvan skip this step.

Installing java

Check the java installation in your system by typing " java -version " in a terminal. It will display the installed java details .

If java is not installed, then install java using the command

sudo apt install openjdk-8-jdk-headless

Setting JAVA_HOME path

Open /etc/environment file using any editor like gedit or nano using the command 
sudo gedit /etc/environment
and add the following lines (your jdk name may differ. To check your jdk name  run 
cd /usr/lib/jvm/
and then run 
ls
to view the jre and jdk folders ) 
JAVA_HOME="/usr/lib/jvm/java-8-openjdk-amd64"
              export JAVA_HOME
Now, to apply the changes, we have to  update /etc/environment .For that run 
source /etc/environment
Now , open  .bashrc  file using the command 
sudo gedit ~/.bashrc
and add the above two lines here also. And update the .bashrc file by running 
source .bashrc
Verify JAVA_HOME by running the command 
echo $JAVA_HOME
. If everything is correct , it will display the JAVA_HOME location.

Configuring ssh

For installing hadoop , we have to configure ssh. First, install ssh by running the command 
sudo apt install ssh
Next, open a terminal and run the following command. 
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
It will generate a rsa key, like below. Now, we have to add that key to authorized keys using the following command. 
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
To check the ssh, run ssh localhost .It must login without asking any password. If it asks for password, then you have to repeat the above ssh configuration steps.

Installing Hadoop in Pseudo Distributed Mode

Download hadoop-2.7.3 from this link. After downloading, extract it to the folder where you want to install hadoop. I installing  it in "/usr/local/hadoop " .  So after extracting the hadoop jar, run the below command to copy it to "/usr/local/hadoop ".  
sudo mv hadoop-2.7.3 /usr/local/hadoop

Step I: Setting up Hadoop

We have to set HADOOP_HOME path . For that open terminal and edit the .bashrc file. 
sudo gedit .bashrc
add the following lines 
#hadoop
              export HADOOP_HOME=/usr/local/hadoop
              export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
refresh the .bashrc file 
source .bashrc

Step II: Hadoop Configuration

Adding HADOOP_PREFIX

Previously, we set JAVA_HOME path. Similarly, we have to set HADOOP_PREFIX path in hadoop-env.sh  . file present in etc/hadoop folder . This is to specify the path for our hadoop folder.  To do that, open a terminal and run the following commands. 
cd /usr/local/hadoop/etc/hadoop
gedit hadoop-env.sh
and, in that file, add the HADOOP_PREFIX path 
export HADOOP_PREFIX=/usr/local/hadoop

Configuring  core-site.xml  file

Open core-site.xml file , which is located in " hadoop/etc/hadoop " and add the following lines. 
gedit core-site.xml

               	
               		fs.defaultFS
               		hdfs://localhost:9000

Configuring hdfs-site.xml file

Open hdfs-site.xml file , which is located in " hadoop-2.7.3/etc/hadoop " and add the following lines. Specify the path where you want to store the dfs data in "dfs.name.dir" and "dfs.data.dir" property 
gedit hdfs-site.xml
               
              	
                		dfs.name.dir
                		file:///home/USERNAME/pseudo/dfs/name
              	
              	
                		dfs.data.dir
                		file:///home/USERNAME/pseudo/dfs/data
              	
              	
                 		dfs.replication
                  		1

Configuring mapred-site.xml file

Open mapred-site.xml file (create if not present ) , which is located in " hadoop-2.7.3/etc/hadoop " and add the following lines. 
gedit mapred-site.xml

               	
               		mapreduce.framework.name
               		yarn

Configuring yarn-site.xml file

Open yarn-site.xml file , which is located in " hadoop-2.7.3/etc/hadoop " and add the following lines. 
gedit yarn-site.xml

               	
               		yarn.resourcemanager.hostname
               		localhost
               	
               	
               		yarn.nodemanager.aux-services
               		mapreduce_shuffle

Running hadoop

To start hadoop, first we have to format the namenode. For that change directory to hadoop installation  folder and run the command 
hdfs namenode –format
If everything is correct it will show the following screen. (dont worry if your screen dont match this ). Now, after formatting namenode , we have to start  namenode and datanode . For that run the command. 
start-dfs.sh
Then to check whether namenode and datanode are running or not, run jps . It will list all the java processes running in the system. You can notice that namenode, datanode and secondarynamenode are running. Now, we have to start yarn. To do that , run 
start-yarn.sh
You will get a screen like this. You can observe that resourceManager and NodeManager are also running. Thats it....Our installation is completed... :) Now, open your browser and run http://localhost:50070 to know about your namenodes and datanodes. You will get  a screen like this. Now,  in another tab, run http://localhost:8088 to manage your cluster. If you want to stop hadoop, then type and run 
stop-dfs.sh
stop-yarn.sh
Or , simply you can run 
stop-all.sh
And, each time you dont need to format the namenode. Just start dfs and yarn using the  above specified commands or, simply running 
start-all.sh
